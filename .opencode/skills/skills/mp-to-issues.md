# To Issues

Break a plan into independently-grabbable issues using vertical slices.

Process:
1. Gather context from conversation
2. Explore codebase optionally
3. Draft vertical slices (tracer bullets through ALL integration layers end-to-end)
4. Quiz user on granularity, dependencies, HITL/AFK assignments
5. Publish issues in dependency order

Vertical slice rules: Each delivers a narrow but COMPLETE path through every layer. A completed slice is demoable on its own. Prefer many thin over few thick.
